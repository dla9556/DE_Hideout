# 1. 메인 허브 (Main Hub) — 은거지(HideOut)

## 1-1. 전체 UI 레이아웃 개요

메인 허브 화면은 **3D 배경 씬(Scene) 위에 HUD 레이어를 오버레이하는 구조**로 구성된다.
UI는 크게 5개 영역(Zone)으로 분리된다.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  [ZONE-A] 상단 탭 바                                            [h: 48px]   │
│                                                                             │
│  🏠 은거지  닉네임·Lv  공방  성장  교환소/상인  길드 사무소  용병  우편함  소식  시스템       │
│                                       🪙 실버  🔷 D토큰  🔰 계곡의 인장  🌐 서버  │
│                                                                             │
├──────────────────┬──────────────────────────────────┬──────────────────────┤
│                  │                                  │                      │
│ [ZONE-B]         │         [ZONE-C] 중앙 3D 씬       │ [ZONE-D]             │
│ 중앙 좌측 패널    │   "PlayerName"  Lv.47            │ 중앙 우측 패널        │
│ (190px)          │         [3D 캐릭터 렌더]           │ (190px)             │
│                  │       (화면을 채우는 크기)          │                      │
│ - 길드 사무소 위젯 │                                  │ - 스쿼드 설정         │
│ - 채팅창         │                                  │   (랜덤 스쿼드 포함)  │
│                  │                                  │ - 플레이 버튼         │
├──────────────────┴──────────────────────────────────┴──────────────────────┤
│  [ZONE-E] 하단 탭 바 (항상 표시 · z-index 최상위)                 [h: 46px] │
│  보관소  캐릭터 변경                                      친구   │
└─────────────────────────────────────────────────────────────────────────────┘
```

### [ZONE-C] 중앙 3D 캐릭터 디스플레이

#### 렌더링 방식

- ZONE-C에서 캐릭터는 **항상 카메라 정면을 향하는 빌보드 방식**으로 렌더링한다.

#### 캐릭터 크기
- 내 캐릭터: 화면을 채우는 크기 (기준 100%)
- 좌우 파티원: 내 캐릭터 대비 90% 크기

#### 인원수별 카메라 · 씬 배치

| 상황 | 카메라 | 배경 씬 |
|------|--------|---------|
| 솔로 (1인) | 캐릭터 정면 미디엄샷, 약간 우측 오프셋 | 허브 기지 내부, 소수 NPC 배치 |
| 듀오 (2인) | 풀샷으로 줌아웃, 두 캐릭터 프레임 인 | 두 캐릭터 나란히 배치, 간격 1.2m |
| 트리오 (3인) | 풀샷 + 추가 줌아웃, 삼각 배치 | 파티장 중앙 전방, 나머지 좌우 후방 배치 |
| 로드아웃 편집 진입 | 카메라 슬로우 달리-인, 상반신 클로즈업 | 배경 블러 처리 (DOF 효과) |

#### 캐릭터 닉네임/레벨 표기
- 3D 캐릭터 렌더 영역 **상단**에 표기 (캐릭터 머리 위가 아님)
- 스쿼드 파티장은 닉네임 옆에 👑 아이콘 표시

**솔로 (1인)**
```
  PlayerName  Lv.47
  ┌────────────────────┐
  │  [3D 캐릭터 렌더]  │
  └────────────────────┘
```

**스쿼드 (2인)**
```
  👑 Teammate_01 Lv.23   PlayerName Lv.47
  ┌───────────┐          ┌───────────────┐
  │ [파티원]  │          │  [내 캐릭터]  │
  │  (90%)    │          │   (100%)      │
  └───────────┘          └───────────────┘
```

**스쿼드 (3인)**
```
  👑Teammate_01 Lv.23   PlayerName Lv.47   Teammate_02 Lv.31
  ┌───────────┐         ┌───────────────┐  ┌───────────┐
  │ [파티원]  │         │  [내 캐릭터]  │  │ [파티원]  │
  │  (90%)    │         │   (100%)      │  │  (90%)    │
  └───────────┘         └───────────────┘  └───────────┘
```

**닉네임 레이블 세부 스펙**

| 속성 | 값 | 비고 |
|---|---|---|
| 표시 위치 | 캐릭터 렌더 영역 상단 | — |
| 표시 내용 | `player.displayName` + `Lv. {level}` | — |
| 파티장 표시 | 닉네임 좌측 👑 아이콘 | `member.isLeader === true` 시 |
| 폰트 크기 | 내 캐릭터 14px, 파티원 12px | — |
| 텍스트 색상 | 내 캐릭터 `#FFFFFF`, 파티원 `#7EC8E3` | — |
| 배경 | 반투명 검정 `rgba(0,0,0,0.45)`, 라운드 4px | — |
| 최대 닉네임 길이 | 12자 초과 시 말줄임 처리 | — |

### 암거래 골목 뷰 — SVG 씬 구성

분위기: **중세 판타지 암거래 골목** — 어두운 보라-흑 배경, 의심스러운 인물, 밀수품

SVG 씬 요소:
- 원근 수렴 구조 (소실점 cx 47%, cy 19%, 좌/우 벽면 수렴)
- 좌/우 벽면 석판 패턴 (수평·수직선, 보라회색)
- 바닥 조약돌 원근 격자선
- 소실점 보라색 대기 글로우
- 가스등 4개 (초록-노랑 glow + CSS 깜박임 `.mc-gl1~4`)
- 중앙 천장 보라 랜턴
- 수배 포스터 2개 (왼쪽 벽)
- 후드 의심스러운 인물 실루엣 2명 (소실점 근처)
- 밀수품 상자/통 (바닥 좌/우)
- 천장 체인 2개
- 하단 보라 안개 · 상하단 비네트

## 6. 보관소 (HUD 오버레이)

진입: ZONE-E 보관소 버튼 → HUD 오버레이

### 패널 구성 순서 (좌 → 우)

| 순서 | 패널 | 설명 |
|------|------|------|
| 1 | 로드아웃 프리셋 | 프리셋 목록 (활성 프리셋 강조 표시) |

| 4 | 장비 슬롯 + 소모품 슬롯 + 배낭 패널 | 장비 슬롯 → 소모품 슬롯(4칸) → 배낭 패널 → 안전 가방 슬롯 (세로 배치) |

### 4-1. 로드아웃 프리셋

프리셋 목록을 표시하며, 활성 프리셋은 강조 색상으로 구분한다.
- 프리셋 #1 (활성) / 프리셋 #2 / + 프리셋 추가

### 4-2. 보관함 필터링

보관함 왼쪽에 **세로 아이콘 바**로 배치. 마우스 오버 시 라벨이 아이콘 오른쪽에 표시.

| 아이콘 | 구분 |
|--------|------|
| 🗂 | 전체 |
| ⚔ | 무기 |
| 🛡 | 방어구 |
| 🧪 | 소모성 |
| 📜 | 레시피 |
| 📦 | 기타 |

### 4-3. 보관함

| 항목 | 내용 |
|------|------|
| 기본 슬롯 수 | 60칸 (4열 그리드, 세로 스크롤) |
| 확장 단위 | +24칸 |
| 확장 비용 | 골드 (1,000 + 확장 횟수 × 500 G) — 1회: 1,000 / 2회: 1,500 / 3회: 2,000 … |
| 확장 방법 | 헤더 [+ 확장] 버튼 → 확장 팝업에서 비용 확인 후 확정 |
| 골드 부족 시 | 확장하기 버튼 비활성 |

### 4-4. 장비 슬롯

4열 그리드 배치. 배낭 슬롯은 하단에 단독 전 너비 행으로 표시.

| 슬롯 | 비고 |
|------|------|
| 주무기 | — |
| 보조무기 | — |
| 헬멧 | — |
| 상의 | — |
| 하의 | — |
| 장갑 | — |
| 신발 | — |
| 목걸이 | 악세서리 슬롯 (1) |
| 반지 | 악세서리 슬롯 (2) |
| 배낭 | 장착 배낭명 표시. 배낭에 따라 배낭 패널 슬롯 수 변동 |

### 4-5. 소모품 슬롯

장비 슬롯 하단에 배치. 4칸 고정 (4열 1행). 확장 여부 미결정.

> 슬롯 확장(칸 수 증가) 여부는 추후 결정 예정.

### 4-6. 배낭 패널

소모품 슬롯 하단에 배치. 장착된 배낭 종류에 따라 그리드 크기 변동.

| 배낭 종류 | 그리드 | 슬롯 수 |
|----------|--------|--------|
| 기본 배낭 | 4×2 | 8칸 |
| 중형 배낭 | 4×3 | 12칸 |
| 대형 배낭 | 4×4 | 16칸 |
| 특대형 배낭 | 4×5 | 20칸 |
| 장착 없음 | — | 0칸 (안내 메시지 표시) |

#### 모두 보관함으로 이동

배낭 패널 헤더 우측 버튼. 배낭 내 모든 아이템을 보관함으로 일괄 이동한다.

| 상황 | 처리 |
|------|------|
| 배낭 아이템 없음 | "배낭에 이동할 아이템이 없습니다" 토스트 |
| 보관함 여유 공간 부족 | 이동 차단 + "보관함 공간이 부족합니다. 필요: N칸 · 여유: M칸" 에러 토스트 |
| 이동 성공 | 배낭 슬롯 비움 + 보관함 사용 칸 업데이트 + 성공 토스트 |

### 4-7. 안전 가방 슬롯

배낭 패널 하단에 배치. 배낭 종류에 따라 슬롯 수 변동. 배낭 미장착 시 0칸으로 섹션은 항상 표시.
슬롯 크기는 배낭 슬롯과 동일한 4열 그리드 기준(셀 1개 크기).

| 배낭 종류 | 안전 가방 슬롯 수 |
|----------|----------------|
| 기본 배낭 | 1칸 |
| 중형 배낭 | 1칸 |
| 대형 배낭 | 2칸 |
| 특대형 배낭 | 2칸 |
| 장착 없음 | 0칸 (섹션은 표시) |

## 9. 매치메이킹 예외 처리

| 상황 | 처리 방식 |
|---|---|
| 파티원 중 `status === IN_GAME` 멤버 존재 | 플레이 버튼 비활성 + "진행 중인 레이드가 있는 파티원이 있습니다" 툴팁 |
| 선택 던전 서버 점검 중 | 핀 비활성 + "점검 중" 레이블 |
| 매치메이킹 타임아웃 (120초) | 자동 취소 + "매치를 찾지 못했습니다" 토스트 |
| 네트워크 연결 끊김 | 매치메이킹 즉시 중단 + 재연결 시도 모달 |
| 로드아웃 무기 미장착 | 경고 팝업 "장비 없이 출격하시겠습니까?" |
| 보관함 공간 0% | 경고 팝업 "보관함 공간이 가득 찼습니다." |
| 보관소 공간 부족 (결과창) | 이동 실패 · 에러 토스트 |
| 작업장 업그레이드 재료 부족 | 업그레이드 버튼 비활성 |
