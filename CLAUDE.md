# D&E 아웃게임 프로젝트

> 기획 기준일: 2026-03-09 | 기획서 원본: `Dosc/design-spec.md`

## 프로젝트 개요

D&E 신규 프로젝트의 **아웃게임(Out-Game) UI 시스템** 구현 프로젝트.
메인 허브(은거지/HideOut)를 중심으로 매치메이킹, 스쿼드 구성, 지역 선택 플로우를 포함한다.

---

## 아키텍처 개요

### 화면 구조

메인 허브는 **3D 배경 씬 위에 HUD 레이어를 오버레이**하는 구조.
5개 Zone으로 분리된다.

| Zone | 위치 | 내용 |
|------|------|------|
| ZONE-A | 상단 탭 바 (h: 72px) | 닉네임, 퀘스트, 작업장, 상인, 성장, 우편함, 재화, 서버 상태 |
| ZONE-B | 중앙 좌측 (320px) | 퀘스트 추적기 위젯, 채팅 위젯 |
| ZONE-C | 중앙 3D 씬 | 캐릭터 렌더, 닉네임 레이블, 스쿼드 표시 |
| ZONE-D | 중앙 우측 (320px) | 스쿼드 설정 위젯, 플레이 버튼 |
| ZONE-E | 하단 탭 바 (h: 48px) | 인벤토리 (HUD 오버레이), 캐릭터 변경, 친구 |

---

## 주요 씬/화면 목록

- `메인 허브 (은거지)` — 기본 로비 화면
- `성장 트리` — 레벨/경험치 관리
- `상인 목록 씬` — 아이템 구매/판매
- `성장 목록 씬` — 성장 관련 콘텐츠
- `인벤토리 씬` — 인벤토리/장비 관리
- `설정 씬` — 게임 설정
- `지역 선택 오버레이` — 플레이 전 지역 선택
- `매치메이킹 모달` — 매치 탐색 및 대기

## 핵심 데이터 구조

### Region (지역)

```typescript
interface Region {
  regionId: string;
  name: string;
  dangerLevel: 'LOW' | 'MED' | 'HIGH' | 'EXTREME';
  playerCountRange: { min: number; max: number };
  isUnlocked: boolean;
  activeEvent: EventInfo | null;
  recommendedGearScore: number;
}
```

### 닉네임 레이블 스펙

| 속성 | 값 |
|------|-----|
| 위치 기준 | 캐릭터 머리 본 상단 + Y: +0.25m 오프셋 |
| 내 캐릭터 폰트 | 14px, `#FFFFFF` |
| 파티원 폰트 | 12px, `#7EC8E3` |
| 배경 | `rgba(0,0,0,0.45)`, 라운드 4px |
| 최대 길이 | 12자 초과 시 말줄임 (`Longnickname…`) |
| Fade-Out 조건 | 근접 클로즈업, 로드아웃 편집 진입 시 |

## UX 플로우

### 플레이 진입 플로우

```
플레이 버튼 클릭
  → 지역 선택 오버레이 (Fade-In 0.3s)
    → 지역 타일 클릭 → 지역 상세 정보 패널
      → [이 지역으로 선택]
        → 로드아웃 선택 화면 (무료 / 커스텀)
          → 매치메이킹 모달
            → 매치메이킹 진행 중 (스피너 + mm:ss 카운터)
              → 매치 성공 → 로딩 화면
```

### 스쿼드 구성

- 기본: off 상태 (솔로)
- 최대: 3인 파티
- 파티장: 닉네임 옆 왕관 아이콘
- 준비 완료 인원: 준비 완료 아이콘 표시
- 3D 씬에서 파티원은 ZONE-B / ZONE-D를 침범하지 않음

## 예외 처리 규칙

| 상황 | 처리 방식 |
|------|----------|
| 파티원 `IN_GAME` 상태 | 플레이 버튼 비활성화 + 툴팁 |
| 선택 지역 점검 중 | 빗금 오버레이 + "점검 중" 레이블, 선택 불가 |
| 매치메이킹 타임아웃 (120초) | 자동 취소 + 토스트 메시지 |
| 네트워크 끊김 | 즉시 중단 + 재연결 모달 |
| 로드아웃 무기 미장착 | 경고 팝업 (출격 선택 가능) |
| 보관함 공간 0% | 경고 팝업 |
| 매치 60초 초과 | "서버 혼잡" 대기 여부 팝업 |

## 구현 상태 (미구현 항목)

> 아래 항목은 클릭 시 **'구현 예정 안내' 메시지**만 출력한다.

- 작업장 탭
- 우편함 탭
- 채팅 위젯 (형태 미정)
- 닉네임 레이블 빌보드 처리 방식 (회전 지원 여부 미결)

## 개발 규칙

- Zone 레이아웃 변경 시 기획서(`Dosc/design-spec.md`) 먼저 확인
- 데이터 필드명은 기획서의 camelCase 명명 규칙 준수 (`regionId`, `dangerLevel`, `playerCountRange` 등)
- 미구현 기능은 UI는 배치하되 기능은 '구현 예정 안내' 토스트/모달로 처리
- 매치메이킹 상태 enum: `IDLE` / `SEARCHING` / `FOUND` / `CANCELLED`
- 서버 상태 enum: `OK` / `WARN` / `ERROR`
- 위험도 enum: `LOW` / `MED` / `HIGH` / `EXTREME`

## AI 응답 규칙

### 언어

- 모든 응답, 결과값, 설명은 **반드시 한국어**로 작성한다.
- 영어 사용 최소화. 기술 용어·고유명사에 한해 영어 병기 허용.

### 구현 전 인터뷰 (HARD-GATE)

> 구현 작업 요청이 들어오면 코드 작성 전에 반드시 인터뷰를 먼저 진행한다.

#### 인터뷰 진행 방식

1. **시작 시** 전체 질문 수를 먼저 공지한다. (예: "총 N개의 질문이 있습니다.")
2. 질문은 **객관식 4지선다 + 주관식 1개** 형식으로 구성한다.
3. 객관식 문항에는 **추천 선택지를 명시**한다. (예: `✅ 추천`)
4. 질문 범위: 기술적 구현, UI/UX, 우려 사항, 트레이드오프 — 뻔하지 않고 심도 있게.
5. 인터뷰가 끝나면 답변을 기반으로 **`plan-spec.md` 파일을 작성**한다.

#### 인터뷰 질문 형식 예시

```
[질문 2/5] UI 전환 애니메이션 전략은?

a) CSS transition 기반 (간단, 성능 보통)
b) requestAnimationFrame 직접 제어 (고성능, 구현 복잡)
c) 애니메이션 라이브러리 사용 — Framer Motion 등 (✅ 추천: 생산성 최적)
d) 애니메이션 없음 (즉시 전환)

주관식) 이 화면에서 가장 중요하게 생각하는 사용자 경험 목표는 무엇인가요?
```

#### plan-spec.md 작성 규칙

- 인터뷰 답변을 반영해 구체적인 구현 계획 명시
- 파일 위치: 작업 디렉터리 루트 또는 `Dosc/` 하위
- 포함 항목: 목표, 기술 스택, 컴포넌트 구조, 데이터 흐름, 예외 처리, 단계별 구현 순서
